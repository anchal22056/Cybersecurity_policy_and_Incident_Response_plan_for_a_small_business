# Security Audit Checklist

## Purpose

This checklist is used to verify that cybersecurity controls, policies, and systems are properly implemented and maintained within the organization.

---

# General Security

- [ ] Security policies are documented.
- [ ] Employees understand cybersecurity policies.
- [ ] Security awareness training is completed.
- [ ] Incident Response Plan is available.
- [ ] Disaster Recovery Plan is documented.

---

# Password Security

- [ ] Password policy is enforced.
- [ ] Passwords contain at least 12 characters.
- [ ] Multi-Factor Authentication (MFA) is enabled.
- [ ] Default passwords have been changed.
- [ ] Passwords are reviewed regularly.

---

# User Access Control

- [ ] User accounts are reviewed regularly.
- [ ] Unused accounts are removed.
- [ ] Administrative access is restricted.
- [ ] Access permissions follow the principle of least privilege.
- [ ] Guest accounts are disabled when not required.

---

# Network Security

- [ ] Firewall is enabled.
- [ ] Router firmware is updated.
- [ ] Wi-Fi network is secured.
- [ ] VPN is used for remote access.
- [ ] Network traffic is monitored.

---

# Endpoint Security

- [ ] Antivirus software is installed.
- [ ] Antivirus definitions are updated.
- [ ] Operating systems are updated.
- [ ] Software patches are installed.
- [ ] USB device usage is controlled.

---

# Data Protection

- [ ] Sensitive data is encrypted.
- [ ] Backups are performed regularly.
- [ ] Backup restoration is tested.
- [ ] Confidential files are protected.
- [ ] Data retention policies are followed.

---

# Email Security

- [ ] Spam filtering is enabled.
- [ ] Email attachments are scanned.
- [ ] Phishing protection is active.
- [ ] Employees know how to report phishing emails.
- [ ] Email security policies are followed.

---

# Physical Security

- [ ] Server room access is restricted.
- [ ] CCTV is operational.
- [ ] Visitor access is monitored.
- [ ] Equipment is physically secured.
- [ ] Emergency exits are accessible.

---

# Incident Management

- [ ] Security incidents are documented.
- [ ] Incident reports are completed.
- [ ] Digital evidence is preserved.
- [ ] Lessons learned are documented.
- [ ] Incident Response Plan is tested.

---

# Compliance Review

- [ ] Security policies are reviewed annually.
- [ ] Audit findings are documented.
- [ ] Corrective actions are implemented.
- [ ] Compliance requirements are met.
- [ ] Management has approved the audit.
