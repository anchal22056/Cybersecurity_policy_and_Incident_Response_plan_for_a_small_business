# Security Policy

Thank you for helping improve the security of this project. We appreciate responsible disclosure of security vulnerabilities.

---

# Supported Versions

The following versions of this project are currently supported with security updates.

| Version | Supported |
|---------|-----------|
| 1.0.x | ✅ Yes |
| Older Versions | ❌ No |

---

# Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

Include the following information in your report:

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Screenshots (if applicable)
- Suggested fix (optional)

---

# Response Process

Once a vulnerability is reported, the project maintainers will:

1. Acknowledge the report within **48 hours**.
2. Investigate and verify the issue.
3. Assess the severity of the vulnerability.
4. Develop and test a fix.
5. Release a security update if necessary.
6. Notify users after the issue has been resolved.

---

# Responsible Disclosure Guidelines

Please:

- Do not publicly disclose vulnerabilities before they are fixed.
- Do not exploit vulnerabilities beyond what is necessary to demonstrate the issue.
- Do not access or modify data that does not belong to you.
- Follow ethical hacking practices and applicable laws.
