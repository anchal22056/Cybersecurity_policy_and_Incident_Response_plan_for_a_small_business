# Incident Response Checklist

## Purpose

This checklist helps the Incident Response Team (IRT) respond to cybersecurity incidents in a structured and organized manner.

---

# Detection

- [ ] Receive incident report.
- [ ] Record the date and time.
- [ ] Identify the affected systems.
- [ ] Determine the incident type.
- [ ] Assess the severity.

---

# Containment

- [ ] Disconnect infected devices.
- [ ] Disable compromised user accounts.
- [ ] Block malicious IP addresses.
- [ ] Preserve logs and evidence.
- [ ] Inform the Incident Response Team.

---

# Investigation

- [ ] Identify the attack source.
- [ ] Review system logs.
- [ ] Collect screenshots.
- [ ] Document affected files.
- [ ] Preserve digital evidence.

---

# Eradication

- [ ] Remove malware.
- [ ] Delete malicious files.
- [ ] Apply security patches.
- [ ] Reset passwords.
- [ ] Scan all systems.

---

# Recovery

- [ ] Restore backups.
- [ ] Verify system functionality.
- [ ] Monitor systems.
- [ ] Confirm normal operations.
- [ ] Close the incident.

---

# Post-Incident Review

- [ ] Prepare incident report.
- [ ] Conduct team meeting.
- [ ] Update security policies.
- [ ] Improve employee training.
- [ ] Document lessons learned.
